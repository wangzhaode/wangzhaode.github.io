合成数据实战指南：从零推演 FinePhrase 百亿 Token 级高质量数据集构建

Created on March 10, 2026

2026   ·   AI   LLM   Synthetic Data

引言

如何通过正确的提示词、模型和基础设施组合，将充斥着噪音和无效信息的网络文本转化为最先进的预训练数据？为了寻找合成预训练数据的最佳“配方”，Hugging Face 团队近期运行了 90 个实验，生成了超过 1 万亿个 token，耗费约 12.7 个 GPU 年。其最终的结晶是一份名为 FinePhrase 的巨型数据集（包含 4860 亿 token）。在多项基准测试中，FinePhrase 的表现显著优于所有现有的合成数据基线。

如果你一直关注大型语言模型（LLM）的最新进展（如 Nemotron 3、Qwen3、Phi-4 等），不难发现合成数据已经悄然成为 LLM 训练不可或缺的核心成分。回顾 LLM 的数据发展史，我们可以清晰地看到三次范式转变，而合成数据正是这场数据演进的新纪元。

数据演进的三个阶段

  1. 从维基百科到万亿 Token 规模：早期的语言模型主要依赖 Wikipedia 等高质量小规模数据集。随后，业界开始大举引入网络抓取数据。C4、The Pile 等数据集将规模推向数百 GB 级别；而更为近期的 FineWeb 和 DCLM 更是将数据底盘带到了万亿 token 级别，几乎囊括了所有可爬取的公开网页内容。
  2. 从追求数量，转向追求质量：当单纯扩展网络数据的“量”逼近极限时，重头戏变成了“质”。研究人员开始大量引入启发式规则和去重策略，甚至动用强大的神经元分类器去挖掘“教育性”和“指令化”数据。例如，FineWeb-Edu 使用 70B 的 Llama 3 来评估内容的教育价值；而在精心清洗的 DCLM 数据上，仅用 2.6T token 就能训练出一个在 MMLU 跑分上达到 64% 准确率的 7B 模型。
  3. 合成数据时代的全面到来：网页文本的自然极限毕竟可见，而研究又再三证明了质量远胜于数量。于是合成数据被正式搬上了台面：它的作用是将原本可能会被分类器丢弃的低质量数据“提纯”升级，从而变相增加了高质量数据的总量。通过大模型生成内容的历史由来已久（比如 Cosmopedia 曾使用 Mixtral 生成了 250 亿 token 的教科书和故事），而在如今的前沿模型训练中，合成 token 的比例已经能与真实抓取数据分庭抗礼。

算力资源分配的根本性倾斜

我们观察到一个明显的趋势：模型训练的底层算力分配正在发生根本性的倾斜。以往绝大部分算力几乎全部砸在模型的迭代训练上，但现在，越来越多的算力被按比例划拨给训练数据的清洗与生成环节。无论在预训练阶段还是后训练（Post-training）阶段，这种重分配带来的杠杆效应都不容小觑。

在后训练阶段，合成数据常常扮演着“能力蒸馏”的关键角色。例如，一个前沿的推理模型可以专门针对复杂推理、指令遵循或工具调用生成一系列优质的训练样例用于蒸馏其他模型。譬如在研发 SmolLM2 时，基础模型虽然在编程和数学方面表现还不错，但在面临类似“你好吗”、“嗨”这样的高频闲聊时却表现拉胯；通过生成一份合成的闲聊语料库对其进行定向微调，这块短板便迅速被弥补了。

拒绝“炼金术”，走向“化学工程”

不过对于当下的从业者来说，如何获取正确的合成数据依然颇具“炼金术”的玄幻色彩。到底该选用多大的模型？哪些提示词改写效果最好？如何高效横向扩展计算？面对大模型自己生成的输出，存在“模型崩溃”（Model Collapse）的风险吗？

事实上，所谓的“模型崩溃”，通常是因为模型陷入了纯粹对自己输出进行封闭训练的死循环。在真实的工业实践中，开发者总是将合成数据与人类真实数据进行比例混合，并在提示生成阶段引入极其多元化的外部参考和新信号，深思熟虑地进行整合。

Hugging Face 团队的这系列探索，就是为了将这门玄学转化为严谨的“化学工程”：通过系统、可复现的对照实验来破除对直觉的盲目依赖。下文我们将详细拆解这一分析过程，重点围绕“重新表述（Rephrasing）”策略、各类模型差异带来的生成结果影响，以及支持千亿级数据处理的工程基础设施实践。

重新表述：化腐朽为神奇的核心机制

在合成数据体系里，“重新表述（Rephrasing）”指的是将抓取到的现成杂乱文档输入到语言模型中，在保留原意的前提下，转换且重构其呈现形式。这听起来简单，但其背后的设计空间十分广阔。一篇乱糟糟的帖子，可以被改写为配有详细示例的深度教材、常问常答的 FAQ，或者直接浓缩为严密的百科知识列表。不同的呈现目标会对模型最终固化的能力产生极其针对性的影响。

在这场合成数据的探究中，团队从三个关键维度展开系统性测试：

  1. 重新表述策略： 到底什么样的指令或转换方式才能切实提升特定的下游能力跑分？
  2. 生成器模型： 制造这些合成数据的“苦力大模型”，究竟是参数规模决定数据质量，还是模型自带的系列偏好更重要？
  3. 源数据质量（基底）： 既然是针对原始文本重新表述，那么初始底料的质量重要吗？生成模型是能在低质文本里“无中生有”，还是单纯放大了已有的优质特征？

实践中，研究团队横跨了从极小体积（270M）到大体量（27B）的生成模型体系（如 Gemma-3、Qwen、SmolLM 等），以 FineWeb-Edu 和 DCLM 等多层次网络清洗语料为操作基底，测试了极大量的生成 Prompt，生成了百亿级别的数据用于消融。对于所有的检验，标准都是通过使用 200 亿 token（20B）微调一个 12 亿参数（1.2B）的架构一致的模型，并拿到十余个权威基准（常识推理、阅读理解、数学等）上进行系统评分比对。

实验核心洞察：哪种“配方”真正奏效？

为了给各类合成数据确定坐标靶心，研究人员首先在统一环境下测试了诸如 DCLM、Cosmopedia、Ultra-FineWeb 等现如今最知名的主流预混数据集。测试结果毫不意外：DCLM 无疑是最强的混合基线基底（这也使得它成为整个实验的“假想敌”），Nemotron-HQ-Synth 和 REWIRE （两者均为应用了多元改写提示词的高质量合成子集）也表现不俗。为了搞懂为何它们如此拔尖，研究者对影响数据生成的几大变量参数进行了精巧的剥茧抽丝。

1. 什么样的提示词策略最好？摒弃简单改写，走向“教学重构”

测试发现，如果指令仅仅是让大模型“按照维基百科风格把这段文字重写一遍”或者进行“顺着续写/摘要提取”，生成的文本最终在下游验证里的收益相当平庸，根本无法对抗使用天然优质的 DCLM 语料。

在测试的十余种提示词方向中，表现出异常强悍泛化能力的，是那些将源文本深度重装为“强教学特征”及结构化逻辑的提示组合：这包括了 faq（多层级问答对构建）、math（附带详尽推理演示的数学解析）、table（信息结构表格化总结）以及 tutorial（具有详细步骤与例子的演练教程）。

这些具有浓厚辅导书与指导属性的数据，极其显著地提振了基底模型的自然语言理解、科学客观知识回忆（如 ARC 榜单）以及阅读理解（SQuAD）的能力。

但在此处，数据工程师务必须要引起警觉：这背后隐藏着一个“常识推理”与“知识回调”极度矛盾的权衡机制。测试显示，哪怕是成绩最好的“教学重构型”合成数据，都会相当明显地折损模型的常识推理能力判断直觉（体现在 HellaSwag 和 PIQA 等评测集的得分跳崖式滑落）。之所以合成词条的整体均分还能胜过原生文本，仅仅是因为它们在硬核知识、推理问答上的涨分幅度盖过了常识流失的亏损。

这也彻底解释了当前模型训练界的一条铁律：纯靠合成数据主导训练几乎必然走向局部退化。只有让强推理逻辑的合成数据与包含复杂大千世界逻辑的优质天然数据进行混合，才能让生成模型不仅“知识渊博”，同时还具备“人类常识”。

2. 规模的迷思：用来洗数据的模型，不是越大越好

在这场实验中得到了一个非常反直觉、却又对整个 AI 工业界极其利好的核弹级结论：为了获得一流的重新表述数据，你并不需要巨大的生成模型。

从微型的 270M 参数规模一路向上横向消融测至 27B 的巨兽级别，结果显示：除了极小体积的模型偶有发挥不稳定外，对于常规的深度重写与知识构筑任务，选用十亿到二十亿（1B - 2B）左右参数规模的模型其实已经完全摸到了能力的天花板边界。强行堆砌到 12B 甚至 27B 参数，在模型实际产出的合成数据上，并未给后续微调带来任何统计学上具备显著性差异的质量跃升。

然而，对比参数大小，不同开源模型系列（家族）的风格取向与训练底蕴，反而有着压倒性的决定影响。在对 SmolLM2、Falcon3、Qwen3、Gemma-3、Granite3、Llama-3.2 六大门派的平行角逐中，体积落在 1.7B 量级的 SmolLM2 取得了不可思议的绝对碾压性优势。究其根因，极大概率是出自 SmolLM2 在原初对齐训练周期中，高强度地进食过大量富含重写规划与复杂逻辑拆分的“指令对齐黄金数据”，这使得它天生就自带优秀的“改写导师”修养模板。

3. 数据混合与伴随组合：底料和骨架之间的耦合魔法

既然谈到合成数据的引入与交融，就必须厘清数据的两个关键基础位面：

  • 被改写的源头语料（Source）：巧妇难为无米之炊。要想通过提示词引出丰富的问答机制和数学推导，你抛入大模型的原文本里就不能全是空洞的垃圾论坛灌水。
  • 混合训练集（Mix）：在真实训练最终大杂烩时，你需要和生成数据打配合混掺在一起的天然基底库。

前文提到过生成过程会损伤大模型的“常识”，实验结果对此开出了一剂对症下药的良方：选用 DCLM 作为高质量的混合补充底仓，效果最佳。 当你的合成目标本身具有极好的质量水准结构时，DCLM 当作定海神针加入混合，能帮模型严密修复所有流失的常识。然而，如果是以抢救废料和拉升底线为导向（即你的输入源本身质量偏弱），具有更高密度教育属性判断分层的 FineWeb-Edu-HQ 语料，反而取代了 DCLM 成为极具互补奇效的搭配选择。

对于提示选择的策略广度问题上，数据工程师也不应过分依赖单一神迹指令。实验论证了：指令设计方向的多元组合（即混合多个不同的 Prompt 生成结果）远比强行引入不同模型系列的多元化更能对准痛点靶心。 在实践中，组合应用大约 3-4 个能够导向教学结构的优质提示词场景片段，便能逼近模型泛化表现的黄金最优点。

深度剖析：那些耐人寻味的微小反馈直觉

团队在进行多维度切片的深度分析期间，还摸索梳理出了一些极为有意思的技术经验反馈现象：

1. 数学逻辑重构中的“失败同样极具生产力（Productive Failure）” 在使用 math 系列提示词对相关内容进行分步解析扩写时，人工抽检查看结果发现：参数受限的大模型有时也会抽风，在解析大段算力过程时留下了少许错漏（错误率大约在 5-15% 间波动）。极其反常规的是：恰恰是这群混合了不完美但逻辑拆分极度详尽细碎的数据，让被训练的模型在最终面临 GSM8K（典型数学评测集）时给出了全场最耀眼的飞跃。 这在现代认知教育心理学上有着绝妙的印证——“生产性错误（Productive failure）”。对于大语言模型的预训练吸收周期而言，一份偶有细微演算瑕疵甚至偏航但是逻辑推理链条（CoT）异常丰满详实的长段数据，对于它形成顽强稳健数学学习鲁棒性的效果，简直碾压了一个短平精快且光鲜亮丽的正确死板答案。不加区分地暴力过滤掉这一类“不完美的长思考输出”，对 AI 而言其实是一种绝命短视。

2. 边际递减严重：规模和套娃并不能引发质变 用算力卷生卷死是存在收益结界的。数据显示，一旦在模型里堆够 50B（500亿）左右的改写优质 token 量后，进一步海量扩充数据的获利幅度以对数线性态势惨烈收窄。更有趣的是，如果想要投机取巧，让大模型对着同一篇原始材料以同样的改写套路“反刍”生成二次套娃迭代，这不仅没有正向帮助，反而严重损伤模型的数据基色（伴随信息的剧烈耗损），得不偿失。

3. 对错字具备免疫抗体，对逻辑背叛深恶痛绝 不要对 Prompt 指令的文字打磨有洁癖，但要对内部逻辑有极简洁癖。如果在改写提示词配置里藏着轻微的错别字或部分病态语法结构缺失，受测模型的输出质量基本稳若泰山毫发无损；但如果你在字里行间植入了充满冲突对抗的逻辑指引要求（比如指令让其详尽分解，同时强要求短小精干快速完结），整个大模型当场面临精神错乱，生成的文本输出结构将产生重度瘫痪。

规模化落地的架构支撑体系建设

真正在现实世界生产以供大语言模型消化的数千亿规模合成文本，不仅仅牵扯的是 AI 算法，这实质上是对顶层系统分发构架的一场高阶拉力考验。研究团队不得不搭建了一个由 vLLM（负责超高压推理层）、DataTrove（充当管道数据排单大总管）和 SLURM（系统资源集群调派）深度有机融合的流水车间。

他们构建了如下三支维度的性能基建系统来保证平权释放算力：

  • 分层推理调优： 面对不同梯度的处理机组，采用全自动伸缩配比。轻量模型取消张量并行（Tensor Parallel）将 Batch size 原地吃满撑大到 128；中等重型模型则按 2-4 路执行切分。
  • 推测解码与预填充拉满： 采用分块级别的预填充机制极尽压榨显存开销。更为关键的是，搭配 推测解码（Speculative Decoding）（配合简单的 n-gram 进行基础预测抓取，将单草稿接受概率打底到 70% 黄金段位运行区间），极大拉高了极限输出天际线。在标准 A100 80GB 的评测环境上，使用被点名表扬的 SmolLM2-1.7B 作为全量干苦力的造假数据机，单卡单节点直接爆砍到了 45.5K TPS 的极限吞吐天花板极值速度。 在这种算力加持下，如果是挂靠百卡集群运转，制作百万亿 token 的终极形态数据集，也仅仅只是按日或周结算的高层算力跑马。这实质上彻底封路了之前认为“生成合成数据时间资金成本浩瀚无垠”的消极悲观猜测断言。
  • 优雅降级与全自动容错防线重构： 过去如果因个别极端损坏的文档或由于节点物理故障陷入死区发生 OOM 或挂起崩溃时，经常会导致整个宏伟的集群批次任务前功尽弃全部重返重跑。由于挂载部署了高频容错断点续传设计思想模块体系基建系统（基于微小至单线程级每个 Slurm 都有独立专属处理路径缓存）。系统如今能以令人不可置信的低毫秒反应延迟截断抛去无效报错单元结构并进行极速微恢复，把高额昂贵开销时间一分不散的稳住保全进度条留档。

终极结晶白金配方：构筑 FinePhrase

最终经过全部长达 90 次排列比对演练试错后的所有收获被融会贯通结晶合二为一。就此打造诞生了全系列最佳配比重铸打样标红：生成造字全领域新标杆模型级产物 FinePhrase。 这份重磅产出的“白金配方明码表”架构清晰如下：

  • 原始信息基底锚定盘：投下了原生天然的 FineWeb-Edu-HQ（70%配比基数）并混合注入了原生粗粝 DCLM 模块（剩余调控占 30% 范围份额），建立全域信息知识基础兜底保障。
  • 提示指令矩阵核心骨架：按金字塔形态切割。最高频比重投放在逻辑树最深的 FAQ 问答系统设计领域（四成兵力）；辅以全方位的解析教学数学系统结构框架拉高上限（三成），随之穿插引入严密梳理的信息表格式分拣处理（两成），最后压入用于长链执行串联逻辑向讲解型综合实战引导序列篇（一成）。全面完美全方覆写所有的学术训练指导重塑逻辑方向重心。
  • 车间处理组引擎主力：全系统重任统一下发押宝至体积极小但表现极端神勇的 SmolLM2-1.7b（跑满配置环境全部潜力的 vLLM，全开推测解码机制释放上限）。
  • 天然中和调和融混比例策略方案：出库的“高度强硬知识合成数据”结晶，被无缝融合打散拌进最高级别的优质原生语望 DCLM 处理槽池结构区。采取等分各半 1:1 的完美交融方案比例重组编队。

过去很多人将制造“合成数据”草率地等价视同为了随意搞台开源模型进行同义词翻译无规则造字游戏拼图造梦的过程。但是通过上述如此繁冗庞杂严谨理性的切片分析抽离梳理体系中，那些被奉为业界固有真知的定律规则大片发生颠倒与翻转逆反崩塌解体重构。 在这之中，“使用模型尺寸体积规模论”被彻底抛弃扫入垃圾堆，向工业界重新指出了利用精简专精轻兵模型干大事低成本可控规模路径明路。
